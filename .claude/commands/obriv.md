Был **обрыв интернета/сессии**. Не переделывай вслепую то, что уже сделано — сначала восстанови картину из «земли» (git/файлы/PR), потом продолжи прерванную нить.

## 1. Реконструируй состояние из ground truth

```bash
git branch --show-current
git status --short
git log --oneline -6
git stash list
```
- Незакоммиченные правки — завершённые или оборванные на полпути?
- Последний коммит — та работа, что шла перед обрывом?

## 2. Сверь с PR и фоновыми задачами

```bash
gh pr list --state open
gh run list --limit 3
```
- PR существует/смержён? Не задвоить. Деплой-run завершился?
- Фоновые задачи (dev-сервер :3008+, build): проверь `*.output`; не плодить дубли портов.

## 3. Целостность недавно записанных файлов (грабля харнесса)

Обрыв может совпасть с битой записью: `NUL`-байт → git видит файл как **`Bin`**.

```bash
git add -A && git diff --cached --stat   # исходник как "Bin" → подозрение
```
Вычистить NUL, пересохранить UTF-8:
```bash
node -e "const fs=require('fs');const f=process.argv[1];const b=fs.readFileSync(f);const n=[...b].filter(x=>x===0).length;console.log(f,'NUL',n);if(n)fs.writeFileSync(f,Buffer.from([...b].filter(x=>x!==0)))" '<путь>'
```
