# HTTPS работает: решение — SSL-вкладка домена в панели Джино

Владелец авторизовал панель Джино, дальше сессия сделала всё сама. Итоговая картина (уточнение гочи):

1. **Прокси Джино терминирует внешние 80 и 443** и форвардит оба на внутренний :80 бокса, передавая **X-Forwarded-Proto**. Внутренний порт один на оба; собственный nginx:443 с certbot-сертом снаружи недостижим (без выделенного IP).
2. **Правильный способ https:** панель Джино → Домены → <домен> → **SSL → «Установить» (бесплатный Let's Encrypt)**. Выпуск ~1–2 мин, продление автоматически **на стороне Джино**. После этого их прокси отдаёт валидный серт домена.
3. **Редирект http→https** на боксе — только по `$http_x_forwarded_proto = "http"` (по `$scheme` — петля, т.к. внутрь всегда приходит http).

Проверено извне (Бокс 1): http → 301 → https → 200 + контент-маркер; все маршруты 200. **DoD M1 в исходной формулировке (https 200 + маркер) закрыт полностью.**

## Для GOTCHAS (заменяет формулировку из прошлого письма)

«Jino-VPS без выделенного IP: внешние 80/443 терминирует прокси Джино → certbot на боксе бесполезен для внешнего 443, `--redirect` даёт петлю. https = LE-серт через панель (Домены → SSL → Установить), редирект на боксе — по X-Forwarded-Proto. Вероятно, применимо и к Сабантую (он сейчас http-only — можно включить https тем же способом за 2 минуты).»
