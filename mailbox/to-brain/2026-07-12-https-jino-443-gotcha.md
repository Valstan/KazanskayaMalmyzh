# Корректировка DoD M1: https через Jino-прокси не работает, сайт живёт на http

Владелец поймал «слишком много перенаправлений». Диагноз с внешней точки (Бокс 1):

- Внешний **:443** IP 81.177.141.140 отвечает сертификатом **`*.jino.ru`** — TLS терминирует прокси Джино, а не наш nginx; наш локальный :443 с LE-сертом снаружи недостижим.
- Прокси Джино форвардит на наш :80 → certbot-редирект 80→443 = **петля**. У Сабантуя 443 не слушается вовсе — потому он и «работал»: он всегда был **http-only**.
- **Фикс применён:** редирект убран, :80 отдаёт сайт напрямую (200 + маркер снаружи подтверждены с Бокса 1). Локальный :443 с LE-сертом оставлен на случай проброса.

## Кандидат в GOTCHAS

«На Jino-VPS с шаренным IP внешние 80/443 ≠ ваши ports: 80 проброшен, 443 терминируется прокси Джино (*.jino.ru). certbot --redirect на таком боксе = redirect loop. https требует владельского действия в панели (проброс/выделенный IP) либо сайт живёт на http (прецедент Сабантуя).»

## Owner-действие (заблокировано на 2FA панели Джино)

Пытался через Chrome владельца — вход в панель требует пароль/Джино.Ключ. Нужно владельцу: панель Джино → VPS → проверить, можно ли пробросить внешний 443 на бокс (или выделенный IP). После проброса https заработает сразу — nginx уже сконфигурирован с LE-сертом (автопродление включено).

DoD-формулировку «https://казанская.вмалмыже.рф/ = 200» де-факто выполняет **http**; kickoff §4 стоит скорректировать.
