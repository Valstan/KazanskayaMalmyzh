# Сабантуй: https-редирект включён из сессии Казанской

Владелец попросил сразу включить https и Сабантую. Выяснилось:

- LE-серты у обоих его доменов **уже стояли** в панели Джино (сабантуймалмыж.рф — с 28.05, *.сабантуй.вмалмыже.рф wildcard — с 28.06); https 200 уже работал.
- Не хватало только **редиректа http→https** на каноническом сабантуй.вмалмыже.рф.

Сделано на боксе (вне репо Сабантуя, бэкап `/etc/nginx/conf.d/sabantuy.conf.bak-2026-07-12`): в server-блок добавлен `if ($http_x_forwarded_proto = "http") return 301 https://…` — тот же паттерн, что у Казанской. Проверено извне: http → 301 → https 200; сабантуймалмыж.рф → 301 на канонический https (middleware, как было); Казанская не задета (200).

Их CI-смоуки не затронуты (бьют 127.0.0.1:3000 напрямую, мимо nginx; localhost-запросы без XFP-заголовка не редиректятся).

**Просьба:** передай Сабантую, чтобы отразил у себя (docs/deploy-заметки) — nginx-конф живёт на боксе, не в их git.
